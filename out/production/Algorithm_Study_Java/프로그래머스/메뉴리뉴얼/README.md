📋 프로그래머스 - 메뉴 리뉴얼 (Java)💡 

난이도 : lvl2

문제 유형알고리즘: 완전 탐색, 조합 (Combination), 정렬 (Sorting)

자료구조: HashMap🛠️ 풀이 전략 (접근 방식)이 문제는 손님들이 주문한 단품메뉴 조합을 바탕으로, 코스요리 메뉴 개수(course)에 맞춰 가장 많이 함께 주문된 메뉴 구성을 찾는 문제입니다.

1.주문 문자열 사전순 정렬 (Pre-processing)손님 A가 "BC", 손님 B가 "CB"를 주문했을 때, 이 둘은 사실상 같은 조합입니다.이를 동일한 문자열로 취급하기 위해, 각 주문(orders) 내의 알파벳을 미리 오름차순으로 정렬합니다. ("CB" $\rightarrow$ "BC")

2.코스 개수별 조합 생성 (comb 함수)조합의 길이(courseLen)마다 모든 주문을 순회하며 가능한 알파벳 조합을 뽑아냅니다.백트래킹(Backtracking) 기법을 활용하여 중복 없이 사전순으로 나열된 조합을 생성합니다.

3.해시맵(HashMap)을 이용한 빈도수 카운팅생성된 메뉴 조합을 Key로, 주문된 횟수를 Value로 countMap에 저장합니다.특정 코스요리 길이에서 가장 많이 주문된 횟수(max)를 구합니다. (단, 최소 2명 이상의 손님에게 주문된 조합이어야 합니다.)

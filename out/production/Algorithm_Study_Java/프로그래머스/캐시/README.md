Deque, Set을 조합해서 풀이가 가능하지만, cache hit를 했을 시, Deque에서 remove() 호출해서 값을 꺼낸 후 맨 앞으로 다시 넣어야하는데, 이 과정이 O(n)이 걸림.

따라서, 조회된 값을 뒤로 보내주는 LinkedHashMap 을 고려 가능. LinkedHashMap은 순서 보장, 주요 연산이 O(1)로 처리가능.

기본값은 값을 넣어준 순서로 순서가 보장되지만, 

선언 시에
new LinkedHashMap<>(cacheSize,0.75f,true); 으로 마지막 인자를 true로 넣을 시 값이 접근된 순서로 보장됨. 첫 인자는 초기 map의 크기, 두번 쨰 인자는 얼마만큼 찼을때 map의 크기를 늘릴건지에 대한 옵션. ex) 0.75f이면 75%가 차면 두배로 크기를 늘림. 보통 기본값인 0.75로 두지만, 마지막 인자 true 값을 설정하기 위해 지정해줌.

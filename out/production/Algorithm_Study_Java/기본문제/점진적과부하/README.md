**문제 유형**: 자료구조 (Stack), 모노토닉 스택 (Monotonic Stack, 단조 스택)

시간 복잡도: $O(N)$

📌 **문제 설명**

스쿼트에 진심인 피트니스 매니아가 매일 최대 중량 기록을 측정하고 있습니다. 그의 목표는 더 무거운 무게를 드는 날을 지속적으로 만들어가는 것입니다.매일의 스쿼트 최대 중량 기록이 정수 배열 weights로 주어졌을 때, 각 날의 중량보다 더 높은 중량을 든 날까지 걸린 일수를 계산하여 배열로 반환하는 solution 함수를 작성하세요. 만약 이후에 더 높은 중량을 들지 못했다면 해당 값은 0으로 설정합니다.


💡 풀이 전략 (접근 방식)
1. 완전 탐색 (Brute Force) - $O(N^2)$각 날짜마다 뒤에 있는 모든 날짜를 순회하며 더 큰 숫자를 찾는 방식입니다. 배열의 길이가 최대 $10^5$이므로, $O(N^2)$ 알고리즘을 사용하면 최악의 경우 약 $10^{10}$번의 연산이 필요하여 시간 초과가 발생합니다.

2. 모노토닉 스택 (Monotonic Stack) 최적화 - $O(N)$스택에 값이 아니라 배열의 인덱스(Index)를 보관하는 것이 핵심입니다. 스택 내부를 현재 중량보다 높은 값을 만나지 못한 날들의 인덱스로 채워나가며, 스택 내부가 항상 내림차순(또는 같은 값) 상태를 유지하도록 관리합니다.
현 풀이는 스택에 배열로 인덱스와 무게 정보를 모두 저장했지만, 인덱스만 저장하고 weights 배열 접근으로 동일하게 풀이 가능. 현 풀이는 직관성을 위해 두 값 모두 저장함.

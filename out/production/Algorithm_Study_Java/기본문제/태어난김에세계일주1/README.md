**문제 유형**: 브루트 포스 (Brute Force), 백트래킹 (Backtracking), DFS (깊이 우선 탐색)

**시간 복잡도**: $O(N!)$ (최악의 경우 N!)

📌 문제 설명

주어진 초기 자산 balance를 가지고 최대한 많은 국가를 순서대로 방문하려고 합니다. 각 국가의 정보는 countries 배열로 주어지며, countries[i][0]은 방문 시 소모되는 비용, countries[i][1]은 방문하기 위해 필요한 최소 자산 조건입니다.최적의 순서로 국가를 방문했을 때, 방문할 수 있는 최대 국가의 개수를 반환하는 solution 함수를 작성하세요.


1. 순열 생성 후 시뮬레이션 방식 (이전 접근)방식: 먼저 가능한 모든 국가의 방문 순서($N!$)를 2차원 리스트에 모두 저장한 후, 다시 루프를 돌며 자산 조건을 검증함.

    **단점**: $N$이 작더라도 모든 경로를 메모리에 유지해야 하므로 공간 복잡도가 불필요하게 증가함.

2. 실시간 상태 전이를 이용한 DFS 백트래킹 (현재 최종 정해)방식: 순열을 따로 저장하지 않고, 재귀 함수(dfs)의 인자로 현재까지 방문한 국가 수(count)와 남은 자산(balance)을 직접 넘겨주며 탐색함.장점:자산 조건(balance >= countries[i][1])을 만족하는 경우에만 다음 재귀로 진입하므로 불필요한 탐색이 자연스럽게 가지치기(Pruning)됨.매 재귀의 시작점마다 maxCountries = Math.max(maxCountries, count);를 통해 즉시 최댓값을 갱신하므로 메모리 오버헤드가 없음.방문 표시(visited[i] = true) 후 재귀가 끝나면 바로 false로 원복하는 구조가 직관적임.
